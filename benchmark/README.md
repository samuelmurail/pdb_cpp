# Benchmark

This directory contains the maintained benchmark workflow for `pdb_cpp`.

The benchmark suite is intentionally limited to four scripts:

- `compare_common_speed.py`: benchmark common structure operations across `pdb_cpp`, `pdb_numpy`, `biotite`, and `biopython`
- `compare_dockq_speed.py`: benchmark DockQ CLI against `pdb_cpp.analysis.dockQ`
- `plot_benchmark_results.py`: plot the CSV generated by `compare_common_speed.py`
- `plot_dockq_benchmark.py`: plot the CSV generated by `compare_dockq_speed.py`

All generated CSV and image files are outputs and are not kept in the repository.

## Prerequisites

```bash
cd /home/murail/Documents/Code/pdb_cpp
```

Use an environment that provides the benchmark dependencies. For example:

```bash
conda activate af_analysis
```

The common benchmark imports these libraries when available:

- `pdb_cpp`
- `pdb_numpy`
- `biopython`
- `biotite`

The DockQ benchmark additionally requires the `DockQ` executable to be available on `PATH`.

## 1. Common benchmark

`compare_common_speed.py` benchmarks a fixed set of operations on a list of input structures and writes one CSV row per backend, file, and operation.

The implementation now prefers each library's built-in operations where they exist:

- native RMSD APIs are used for `rmsd_ca_shift`
- native dihedral helpers are used when available
- native sequence-based alignment APIs are used for `align_seq_chainA`
- native spatial search APIs are used for `select_within10_chainA`

For mutating alignment functions, structures are reloaded inside the timed call so one run does not affect the next.

Current operations:

- `read`
- `write`
- `select_within10_chainA`
- `get_aa_seq`
- `rmsd_ca_shift`
- `dihedral_ca`
- `align_seq_chainA`
- `align_ca_self`
- `hbond`

For RMSD and alignment operations, each input structure is transformed once
outside the timed region using a deterministic random rotation and translation.
The benchmark then compares the original file against that transformed partner:

- `rmsd_ca_shift` measures CA RMSD between the original and transformed structures
- `align_seq_chainA` measures sequence-based CA alignment between original and transformed chain A
- `align_ca_self` measures structure alignment between the transformed and original structures

Default input files are defined inside the script:

- `tests/input/1y0m.pdb`
- `tests/input/2mus.pdb`
- `external/8UXA.cif`

### Run with the default structure list

```bash
python benchmark/compare_common_speed.py --runs 5 --warmup 1 --csv external/benchmark_common.csv
```

### Run with a custom structure list

```bash
python benchmark/compare_common_speed.py \
  --runs 5 \
  --warmup 1 \
  --files tests/input/1y0m.pdb tests/input/9X0F.cif \
  --csv external/benchmark_common.csv
```

### Plot the common benchmark

```bash
python benchmark/plot_benchmark_results.py \
  --input external/benchmark_common.csv \
  --output external/benchmark_common.png
```

## 2. DockQ benchmark

`compare_dockq_speed.py` compares the DockQ CLI with `pdb_cpp.analysis.dockQ`.

The default benchmark pairs are defined in the script and include:

- `1rxz_colabfold_model_1.pdb` vs `1rxz.pdb`
- `model.pdb` vs `native.pdb`
- `1jd4.pdb` vs `5m6n.pdb`

### Run the DockQ benchmark

```bash
python benchmark/compare_dockq_speed.py \
  --runs 5 \
  --warmup 1 \
  --mode end-to-end \
  --csv external/benchmark_dockq.csv
```

`--mode cached` reuses preloaded `Coor` objects for the `pdb_cpp` side of the benchmark.

### Plot the DockQ benchmark

```bash
python benchmark/plot_dockq_benchmark.py \
  --input external/benchmark_dockq.csv \
  --output external/benchmark_dockq.png
```

## Output files

Typical outputs are:

- `benchmark_common.csv`
- `benchmark_common.png`
- `benchmark_dockq.csv`
- `benchmark_dockq.png`

These files can be written anywhere by changing the `--csv`, `--input`, and `--output` arguments.
